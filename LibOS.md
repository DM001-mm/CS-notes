# 必要性：

有的应用发现通用操作系统的某些硬件调用 还不如自己的，于是操作系统就将更加底层(接近硬件调用)的api，直接交由用户态的应用来实现，这就抽象出来了 LibOS (库操作系统)

因为内核 实现 也是有 模块化，抽象，[[分层]]，[[层级]] 这些东西，耦合度高，轮到你应用使用的操作系统的api 都不知道是第几层的，所以，有的时候不如应用自己来
# 与操作系统的关系

> 操作系统内核 负责 实现硬件资源在多个LibOS 之间的多路复用，并管理这些LibOS实例的生命周期。

前半句= 内核管理硬件资源，对多个LibOS进行资源管理
后半句= 
```
LibOS = class
App+LibOS = Object
kernel = main
```
由此易知: LibOS 可以自己写，也可以使用别人的
