# 为什么

## 颠簸现象
### 产生原因
页替换算法 与实际负载(进程访问物理页面的状态) 不匹配

**细说**
页替换算法 是 为了尽可能接近MIN算法，但是都建立在一定的假设的基础上的:

| 策略    | 它默认相信什么          |
| ----- | ---------------- |
| LRU   | 最近用过的页，接下来大概率还会用 |
| FIFO  | 最早进入内存的页，可能最不需要了 |
| CLOCK | 用访问位近似判断最近是否用过   |
| LFU   | 用得次数多的页，以后还会继续用  |
|       |                  |

但是实际上，进程访问物理页面(va->pa)，没有按照他的假设来，导致了 一直缺页 一直CPU一直在等待 磁盘，然而 这个时候 ，CPU的调度器 开始发力了，心想 md 我不能闲下来 ，但是因为进程访问的 物理页面 的规律 和 他页替换算法 假设 不匹配，导致 缺页 和等待现象更加严重。 所以，就有了 工作集 模型

# 介绍

一个应用进程 在时刻$t$的工作集$W$ 为它在时间区间$[t-x,t]$使用的内存页集合，也被视为他在未来(下一个x时间内) 会访问的内存集合

早期极端一点的:all-or-nothing
