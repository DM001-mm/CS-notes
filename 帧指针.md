
# 必要性
> 即为什么地址要用 FP的偏移量 而不是 栈地址？

因为 函数的主要作用是为了复用，它的地址只有 在每次运行到它，也就是创建函数实例的时候才会知道自己的地址(帧指针-FP)，那我编译的时候怎么办呢？函数里面的各种语句，参数，变量，我应该怎么编译呢？就需要我们的相对地址来了，定义一个变量 作为参考，然后 一个函数中的任何东西，都相对于 这个变量 编址，然后 别调用时，直接给这个变量赋值就行了，这样很方便

> 如果我们知道了 一个函数的帧指针位置，就可以通过偏移量寻址来访问这个函数管理的全部[[局部状态]]

# 重要性

在递归中 旧的FP 就保存在 旧的栈帧里面，保证每次活跃的 FP 只有一个(FP寄存器只有一个，其他的实际上是ISA规定的在函数调用时，创建了访存调用的语句，存在了内存里面)，有了FP，就可以进行栈追踪，(因为旧的FP 也有相对新的FP的偏移量，就和 树里面的子结点只用记住自己的父节点，就可以一直遍历一整条链)

这对调试于漏洞修复非常重要，栈追踪 可以给出函数调用链，调试器 也能顺着  调用链 找错，在找到错误的时候，只需要追溯错的栈帧的祖先 结点，就能给出错误的位置。

# 缺陷

对运行没有好处，只能用到 调试里面 ，很多追求性能的项目 直接 配置编译器 给他禁了
