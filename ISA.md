> 机器指令的 格式 ，行为 及 处理器的 执行中的 状态有一个规范 ，成为处理器的指令集架构

> ISA 是 处理器向软件提供的接口

初看这两句 ，觉得 这两个联系不上

再看

**ISA 既是CPU的规范，也是软件的规范**

为什么 这样说？(通常汇编程序只能被汇编成一份机器指令，这里拿汇编指令来讲)

add w0 w1 w2

->w0+w1=w2
为什么cpu会理解成这样，而不是w0=w1+w2 ,就是 ISA 对 cpu的行为上的规范

对软件，ISA 规定了 最简单的 机器指令，但是软件一般用到的 指令是 由 这些基础指令 排序，组合而来的，ISA 就需要定义格式，软件不能自己瞎搞，不然 cpu 不认，
```
int c= a+b;
```
可能编译器A——>汇编，mov,add,顺序a
编译B——>汇编，add,mov,........, 顺序b


这样规范<-> 接口，cpu->软件，逻辑上很容易理解了
