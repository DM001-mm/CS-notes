 
# 1. 存在的意义

应用程序不能“乱搞”，否则一个普通程序就可能随便读写内存、修改磁盘、关闭中断、控制外设，甚至破坏其他程序和操作系统本身。

所以操作系统需要把 CPU 的运行状态分成两类：

- **用户态**：普通应用程序运行的状态，权限较低。
    
- **内核态**：操作系统内核运行的状态，权限最高。
    

可以把它理解成：应用程序生活在一个有法律约束的社会里，用户态就是普通人活动的区域，内核态就是执法和管理机构所在的区域。普通人不能直接进入警察局修改法律，也不能直接调度全城资源，必须通过规定的流程申请。

这个“规定的流程”，在操作系统中主要就是 **系统调用**。

---

# 2. 用户态和内核态的权限区别

在用户态下，程序不能执行某些危险操作，比如：

- 直接访问任意物理内存；
    
- 直接操作磁盘、网卡、键盘等外设；
    
- 修改页表；
    
- 关闭中断；
    
- 修改 CPU 的关键控制寄存器；
    
- 随意切换进程。
    

这些操作只能由内核态完成，因此被称为 **特权操作** 或 **特权指令**。

如果用户态程序直接执行这些操作，CPU 会认为它越权了，于是触发异常，把控制权交给操作系统内核处理。

---

# 3. 用户态如何请求内核服务

虽然应用程序权限低，但它确实需要使用操作系统服务，比如：

- 读写文件；
    
- 创建进程；
    
- 申请内存；
    
- 网络通信；
    
- 获取当前时间；
    
- 输出到屏幕。
    

这些事情不能让应用程序直接操作硬件完成，而是通过 **系统调用** 请求内核代劳。

例如：

```c
read(fd, buf, size);
write(fd, buf, size);
open(path, flags);
fork();
```

这些函数表面上是普通函数，但底层会通过特殊指令进入内核态。

---

# 4. 系统调用为什么叫“陷入”

系统调用是一种主动进入内核的方式，通常叫做 **陷入 trap**。

它不是程序出错了，而是程序故意执行一条特殊指令，请求 CPU 从用户态切换到内核态。

大致过程是：

1. 用户程序执行系统调用指令；
    
2. CPU 从用户态切换到内核态；
    
3. CPU 跳转到操作系统提前设置好的系统调用入口；
    
4. 内核根据系统调用号判断用户想要什么服务；
    
5. 内核检查参数是否合法；
    
6. 内核执行对应服务；
    
7. 执行完后返回用户态，应用程序继续运行。
    

所以系统调用可以看成是：应用程序不能直接干某件事，于是走正规流程，向操作系统提出申请。

---

# 5. 异常、中断、陷入的区别

这里容易混。

广义上，CPU 从当前程序跳到内核处理某件事，都可以看成控制权转移到内核。

但原因不同：

### 1. 中断 interrupt

中断通常来自外部设备，是异步发生的。

比如：

- 键盘输入；
    
- 鼠标移动；
    
- 磁盘读写完成；
    
- 时钟中断到来。
    

程序本来正在运行，外设突然通知 CPU：“我这里有事要处理。”

### 2. 异常 exception

异常通常是当前程序执行指令时产生的问题。

比如：

- 除以 0；
    
- 缺页异常；
    
- 非法指令；
    
- 用户态执行特权指令；
    
- 访问非法地址。
    

异常不一定都是坏事。比如缺页异常可能只是说明当前页面还没调入内存，操作系统处理完之后程序还能继续运行。

### 3. 陷入 trap

陷入通常是程序主动触发的，最典型的就是系统调用。

它不是意外，而是程序主动请求内核服务。

所以可以总结成：

- **中断**：外部设备打断 CPU；
    
- **异常**：程序执行过程中出了某种情况；
    
- **陷入**：程序主动进入内核，请求服务。
    

---

# 6. 用户态切换到内核态时发生了什么

用户态进入内核态，不只是简单地“跳转到另一个函数”。

CPU 和操作系统会做很多保护工作：

1. 保存当前用户程序的部分现场，比如程序计数器、状态寄存器等；
    
2. 切换 CPU 的特权级，从用户态变成内核态；
    
3. 切换到内核栈，避免继续使用不可信的用户栈；
    
4. 跳转到内核中固定的入口地址；
    
5. 内核读取系统调用号和参数；
    
6. 检查用户传入的地址、权限和参数是否合法；
    
7. 执行真正的内核服务；
    
8. 返回用户态，恢复用户程序继续执行。
    

这里需要注意：  
**用户态到内核态的切换，不一定等于进程切换。**

如果只是一次普通系统调用，比如获取时间，内核处理完可能马上返回原来的进程。

但如果系统调用导致进程阻塞，比如读磁盘、等网络数据，那么操作系统可能会进行调度，切换到另一个进程运行。

---

# 7. 为什么不能让应用程序直接访问硬件

如果应用程序可以直接访问硬件，就会有很多问题。

比如：

一个程序可以直接修改磁盘，就可能把别人的文件删掉。  
一个程序可以直接修改内存，就可能偷看别的进程的数据。  
一个程序可以关闭中断，就可能让整个系统卡死。  
一个程序可以随便控制 CPU 调度，就可能让其他程序永远得不到运行机会。

所以操作系统必须把危险能力收回到内核中，由内核统一管理。

---

# 8. 总结

内核态和用户态的本质，是一种 **保护机制**。

用户态限制普通程序的权限，防止它破坏系统。  
内核态拥有完整权限，负责管理硬件和系统资源。  
应用程序如果需要使用硬件或系统资源，不能直接操作，而是通过系统调用进入内核，请求操作系统代为完成。

所以用户态和内核态存在的意义就是：

**用硬件提供的权限隔离机制，把普通程序和操作系统内核隔离开，让系统既能运行应用程序，又能保证安全、稳定和资源可控。**
