**重定向**就是：

> 程序本来要从键盘读、往屏幕写；  
> 现在我让它改成从文件读、往文件写。

也就是改变程序的**输入/输出方向**。

---

正常情况下，一个程序有三个默认通道：

```cpp
cin   -> 标准输入  stdin   -> 默认来自键盘
cout  -> 标准输出  stdout  -> 默认输出到屏幕
cerr  -> 标准错误  stderr  -> 默认输出到屏幕
```

可以理解成：

```text
键盘  --->  程序  --->  屏幕
          cin       cout/cerr
```

---

## 1. 输出重定向：`>`

比如程序里有：

```cpp
cout << "hello" << endl;
```

正常运行：

```bash
./program
```

屏幕显示：

```text
hello
```

但是如果你这样运行：

```bash
./program > output.txt
```

意思是：

> 不要把 `cout` 的内容显示到屏幕了，写进 `output.txt` 文件。

所以屏幕上可能什么都没有，但是 `output.txt` 里有：

```text
hello
```

这就是**输出重定向**。

---

## 2. 输入重定向：`<`

比如程序里有：

```cpp
int x;
cin >> x;
```

正常情况下，它会等你从键盘输入。

但如果你有一个文件：

```text
input.txt
```

里面写着：

```text
123
```

然后运行：

```bash
./program < input.txt
```

意思是：

> 不要从键盘读了，从 `input.txt` 里面读。

于是：

```cpp
cin >> x;
```

读到的就是 `123`。

---

## 3. 错误输出重定向：`2>`

重点来了，`cout` 和 `cerr` 是两个不同通道。

```cpp
cout << "正常结果" << endl;
cerr << "错误信息" << endl;
```

如果你运行：

```bash
./program > output.txt
```

这只会重定向 `cout`，也就是标准输出。

结果：

```text
output.txt 里面有：正常结果
屏幕上仍然显示：错误信息
```

因为 `cerr` 走的是另一个通道：`stderr`。

如果你想把错误信息也写进文件，要这样：

```bash
./program 2> error.txt
```

这里的 `2` 表示标准错误流。

---

## 4. 为什么是 `2>`？

因为操作系统给这三个通道编号：

```text
0  -> stdin   标准输入
1  -> stdout  标准输出
2  -> stderr  标准错误
```

所以：

```bash
./program > output.txt
```

其实省略了 `1`，完整写法是：

```bash
./program 1> output.txt
```

而：

```bash
./program 2> error.txt
```

就是：

```text
把 stderr 重定向到 error.txt
```

---

## 5. 一个完整例子

假设代码是：

```cpp
#include <iostream>
using namespace std;

int main()
{
    cout << "正常输出" << endl;
    cerr << "错误输出" << endl;
    return 0;
}
```

直接运行：

```bash
./program
```

屏幕显示：

```text
正常输出
错误输出
```

运行：

```bash
./program > out.txt
```

结果：

```text
屏幕显示：错误输出
out.txt 里面：正常输出
```

运行：

```bash
./program 2> err.txt
```

结果：

```text
屏幕显示：正常输出
err.txt 里面：错误输出
```

运行：

```bash
./program > out.txt 2> err.txt
```

结果：

```text
out.txt 里面：正常输出
err.txt 里面：错误输出
```

---

所以你可以这样记：

```text
重定向 = 改变输入/输出流的去向
```

常见符号：

```bash
>      把标准输出写到文件，会覆盖原文件
>>     把标准输出追加到文件末尾
<      从文件读取标准输入
2>     把标准错误写到文件
```

放到你刚才的问题里：

```cpp
cerr << "Usage: " << args[0] << " HOST PATH\n";
```

它用 `cerr`，是因为这不是程序的正常结果，而是**错误提示**。这样用户可以把正常输出和错误信息分开处理。
